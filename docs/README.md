## üìã Documentation Standards

### When to Create New Documentation
- **Feature Documentation**: When implementing major features
- **API Documentation**: When adding new API endpoints
- **Status Reports**: After completing major milestones
- **Audit Reports**: After codebase reviews or refactoring

### When to Archive Documentation
- When superseded by newer versions
- When features are completed and no longer actively maintained
- When status reports become outdated

### File Naming Conventions
- Use `SCREAMING_SNAKE_CASE.md` for documentation files
- Use descriptive names that indicate content
- Date-specific files should include dates in the name
